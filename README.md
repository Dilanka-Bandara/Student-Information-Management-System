# Student Information Management System

## 📖 Description

A web-based application designed to manage student information, utilizing various enterprise Java technologies. This project was completed as part of the **EC9590: Network Application Design** course for the Department of Computer Engineering at the Faculty of Engineering, University of Jaffna.

## 🏛️ Architecture

The system follows the **Model-View-Controller (MVC)** architectural pattern to separate concerns and ensure the codebase is organized and maintainable.

* 
**Model:** Represents the data and business logic, such as the `Student.java` class.


* 
**View:** The user interface and presentation layer, consisting of JSP pages and HTML forms.


* 
**Controller:** Servlets that manage user input and mediate between the Model and the View.



## 💻 Technologies Used

* 
**Frontend:** HTML, CSS, JavaScript, and JSP (JavaServer Pages).


* 
**Backend:** Servlets for request/response handling and Enterprise JavaBeans (EJB) for business logic.


* 
**Database Interaction:** JDBC (Java Database Connectivity) for database connections and JPA (Java Persistence API) for object-relational mapping.


* 
**Database:** Relational database management systems such as MySQL, Oracle, or PostgreSQL.


* 
**Application Server:** Apache Tomcat for deploying and running the application.



## ✨ Key Functionalities

The application includes the following core CRUD features:

* 
**Student Registration (Create):** Adding new student records including details like name, RegNo, contact information, field of study, and date of birth.


* 
**View Details (Read):** Retrieving and displaying individual or all student records in a structured table format.


* 
**Update Information (Update):** Modifying existing student details using their unique RegNo.


* 
**Delete Records (Delete):** Removing student profiles from the system entirely.
